- `panel-carousel.tsx` uses a reflow-trick (`void track.offsetHeight`)
  to switch between slide/fade modes without animating the transition

### Inertia persistent layouts

The starter kit's `dashboard.tsx` + `settings/{profile,password,appearance}.tsx`
all wrapped their return JSX in `<AppLayout>…</AppLayout>` inline.
That remounts `AppLayout` (and ThemeProvider, sidebars, all state) on
every Inertia navigation, making LHS nav clicks look like full page
refreshes. Fixed by converting each page to the persistent-layout
static form:

```tsx
Dashboard.layout = (page: ReactNode) => <AppLayout>{page}</AppLayout>;
```

Inertia now swaps only inner page content; sidebars and all carousel
state persist across nav. The `<main key={usePage().url}>` in the
layout still re-runs the 200ms `page-fade-in` animation per page.

**Welcome stays opt-out** — `/` is the guest landing page with its own
full markup and should not be inside the DCS shell. "Home" in the
LHS nav now points at `/dashboard`, not `/`.

### Fixed background image — two attempts, one fix

First attempt: `background: url(...) center/cover fixed` on `.hero-bg`
and `.bg-image-section`. The image didn't render at all. Root cause
was unclear — shorthand parsing + `background-attachment: fixed`
breaking inside any ancestor that creates a stacking context.

Second attempt: a separate `position: fixed; inset: 0; -z-10` div at
the layout root. Classic negative-z-index trap — the div rendered
**behind** the body's `bg-background` fill and was invisible.

**Third attempt, which works:** downloaded
`public/Server_Room_Dark.webp` (27 kB) and applied it to `body` via
a scoped class:

```css
body.dcs-shell {
    background-image: url('/Server_Room_Dark.webp');
    background-size: cover;
    background-position: center;
    background-attachment: fixed;
}
```

Added via `useEffect` in `LayoutContent`:

```tsx
useEffect(() => {
    document.body.classList.add('dcs-shell');
    return () => document.body.classList.remove('dcs-shell');
}, []);
```

Browsers **special-case body backgrounds** for
`background-attachment: fixed` — it works reliably there but falls
apart on descendant divs. Scoping via `dcs-shell` keeps welcome/login
pages clean. `.hero-bg` and `.bg-image-section` are now transparent
containers whose `::before` overlays (0.6/0.5 alpha black) dim the
fixed body image. `.content-section` has its own opaque
`--scheme-bg-secondary` and hides the image in between.

## Gotchas worth remembering

- `background-attachment: fixed` **only** works on `body`/`html`
  backgrounds reliably. On a descendant div it breaks as soon as
  anything upstream has a transform/filter/perspective or creates a
  stacking context — which includes your own negative-z-index hack.
- Browser password managers autofill OLDER saved entries for a URL.
  If the field shows 8 dots but your password is 6 characters, the
  autofill is what's being submitted, not what you think you typed.
  The eye-icon toggle is genuinely useful for catching this.
- Inertia pages that wrap their JSX in a layout component **remount**
  the layout on every navigation. Always use the `Page.layout = ...`
  static form for shared shells. The Laravel React starter kit ships
  with the wrong pattern.
- `/usr/sbin/bun` on cachyos can't resolve `node_modules/.bin/vite`
  through `bun run dev`. Use `npx vite` / `npx vite build`
  directly, or run bun via its full path after fixing `~/.bun`.
- dcs.spa's token naming is unprefixed (`--accent`, `--border`).
  Shadcn uses the same names for different things. Prefix DCS
  tokens as `--scheme-*` to avoid collision — everything else can
  match dcs.spa byte-for-byte.

### lucide v1 dropped brand icons

The build cratered with `"Github" is not exported by lucide-react`.
Brand icons (GitHub, Twitter/X, Discord, Apple, etc.) were removed
in lucide v1 for two reasons: **trademark exposure** (each brand has
its own usage policy and the lucide maintainers got tired of fielding
that on behalf of an MIT-licensed library) and **design consistency**
(brand marks don't follow lucide's 24x24 / 2px outline visual grammar).
The recommended replacement is `simple-icons`, but for one-off usage
I wrote an inline SVG at
`resources/js/components/icons/github-icon.tsx` (official Octocat
path) and swapped the three usages in `dashboard.tsx` and
`nav-panel.tsx`. Lighter than adding a dep.

### Vite 8 dev-server IPv6 binding

After the upgrade the browser showed a black screen at
`http://127.0.0.1:8000/`. Vite 8 changed the default server host to
IPv6-only `[::1]:5173` while `php artisan serve` is IPv4 only. The
HTML referenced `http://[::1]:5173/@vite/client` and the browser
couldn't reach it. Fix: pin `server.host: '127.0.0.1'` in
`vite.config.js`. Also removed the now-dead `esbuild: { jsx: 'automatic' }`
block — Vite 8 switched the default transformer to oxc and warns that
both-were-set.

There's a secondary rough edge in Vite 8's config HMR: editing
`vite.config.js` while the server is running triggers a reload that
crashes with `Package subpath './internal' is not defined`. A full
restart (`Ctrl+C composer dev`; run it again) works. First-time boot
is clean; the issue is only with the reload-on-config-change path.

## Sharp edges caught

- **`git-filter-repo` resets the working tree.** It rewrites HEAD
  and then checks out the new HEAD, which **discards any
  uncommitted edits to tracked files**. Untracked files and
  gitignored dirs (`vendor/`, `node_modules/`) are safe. Next time:
  stash or commit before running filter-repo. I didn't see it
  coming; rsync backup at `~/.gh/laradcs-bkp` saved the session.
- Laravel's `composer.json` had a `dev` script calling `bun run dev`.
  On cachyos the user's `~/.bun/bin/bun` shim is a dead symlink, so
  even though `/usr/sbin/bun --version` works, `bun run dev` fails
  to resolve `vite` from `node_modules/.bin`. Switched the script
  to call `npx vite` directly — works unconditionally.
- Browser password managers autofill older saved entries even when
  the displayed placeholder is different. Caught via the eye-icon
  toggle on login: the field showed 8 dots but `passwd` is 6 chars,
  so the browser was filling in something else entirely. Moved the
  default seed password to `password` to match what was actually
  saved.
- lucide v1's brand-icon removal was the single most impactful
  breaking change of the day; worth remembering whenever we bump
  lucide in the future.
- Vite 8's `background-attachment: fixed` bug wasn't real; the
  image-fixed issue earlier in the day was the usual "fixed
  attachment on a descendant div inside a transformed ancestor
  silently clips to zero" gotcha, fixed by moving the image onto
  `body.dcs-shell`.
